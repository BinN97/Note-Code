# 摘要
信息化技术和数字化技术的发展，数字信息的数量增加，但也带来了数字信息的版权保护难题。

传统的版权管理平台使用中心化的数据库架构，存在效率低下，成本高，泄露隐私，安全性不足的问题。

区块链的特性，溯源性、不可篡改、去中心化和分布式存储等，符合需求。

基于Hyperledger Fabric区块链框架，SimHash文本匹配算法，PHsah图像匹配算法，对版权登记进行原创性检测，使用IPFS来存储数据内容。

分析需求，设计功能模块，提供前端界面，是解决版权问题的一种可行方案。

# 绪论
## 背景
信息化技术和数字化的发展，数字信息量持续增加，数字产业内容井喷式增长，发展迅速。但是相应的版权争端也随之增加，互联网数字内容侵权亟需保障。

传统的版权管理平台，采用中心化服务器，缺乏透明度，数据安全难以保证。同时版权分散在各出版商、影视公司中，版权分享意愿低，抑制了所有者的报酬。

版权管理平台存在数据壁垒，数据库无法共享，进行确权时，可能无法追溯原始作者，引起纠纷。同时在录入数据时，缺乏有效监管，可信度得不到保障，被攻击会有数据泄露的风险。

区块链采用无需第三方的中心化理念，通过PoW共识算法和激励机制来确保透明度和安全性，将数据存在区块中，区块之间通过Hash算法保证不可篡改性，分布式节点共同维护数据安全。Hyperledger Fabric项目实现数据分离。

基于此，提出一种基于区块链的数字版权管理系统，解决机构公信力不足、数字版权易受攻击、平台之间无法数据共享等问题。

## 数字版权保护研究现状
数字版权管理技术（Digital Rights Management，DRM），利用相关技术手段防止以数字形式展示的内容（包括文本、图像以及视频等）被非法使用，被认为是解决数字内容在互联网上无序传播的关键技术。

微软推出的Windows Media DRM，保障内容在计算机上安全地传递和播放；国内方正集团为信息传播中涉及的资源数字化、网络出版、数字图书馆以及电子公文传输等领域，提出一种Apabi解决方案；刑建国等人基于PKI和P2P模式设计一种DRM系统；孟宪明等人则基于 P2P 提出了另一种 DRM 系统设计；赵蕴华等人基于 DOI 进行了数字版权管理系统地模型研究；刘静研究了文本水印的数字作品侵权追踪。

### 区块链数字版权管理研究现状
目前数字版权管理方面存在的问题：
+ 确权难：版权登记主要通过版权局和版权中心，申请、审查、确认，步骤繁琐，有着严重的滞后性
+ 取证难：网络信息易于篡改，给侵权的认定带来困难，删除侵权内容将难以核实侵权行为
+ 赔偿界定难：侵权带来的损失难以估算，追究成本过高导致追究过程不顺

区块链的优势：
+ 易确权：将原创内容、原创作者、时间戳等信息存储到区块中，方便迅速，降低成本
+ 验证内容使用权：使用密码学技术，将版权信息写入到数字内容中，需要访问需要私钥
+追溯版权记录：区块链具有不可篡改和可溯源性，数字内容版权信息变更可察觉，为侵权提供证明
+使用成本低：将法律条文写到智能合约中，用户无需相关法律知识

## 研究内容
1. 对数字版权管理平台进行分析，设计新型数字内容版权管理平台
2. 针对区块链无法存储大容量数据的不足，构建基于IPFS的区块链的存储网络
3. 在确权之前进行内容重复度检测，保证内容原创性
4. 实现系统，模块功能
